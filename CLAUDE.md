## Project Purpose

TravelGuide is a menu-driven CLI application, currently a starter template generated by project-creator tooling. Its intended purpose: crawl a specific web page, collect certain kinds of information, and store it in a database. The crawler logic is not yet implemented — `TravelGuide/TravelGuideTaskRunner.cs` (`Run()` throws `NotImplementedException`) is the designated entry point for it.

Code comments are written in Georgian; the user typically communicates in Georgian.

## Critical: External Sibling Dependencies

`TravelGuide.slnx` references ~25 shared library projects that live OUTSIDE this repository, as sibling directories of the repo root (`D:\1WorkDotnet\TravelGuide\`): `AppCliTools`, `ConnectionTools`, `DatabaseTools`, `ParametersManagement`, `SystemTools`, `ToolsManagement`, `WebAgentContracts`. They must be checked out side-by-side or the solution does not build. Key ones:

- **AppCliTools.CliTools / CliMenu** — the CLI app loop (`CliAppLoop`), menu framework (`CliMenuSet`, `CliMenuCommand`), menu-builder services
- **ParametersManagement.LibParameters** — JSON parameters-file management (`ParametersManager`, `IParametersManager`, `ParametersEditor`)
- **SystemTools.SystemToolsShared** — common helpers (`StShared`, `ItemData`, `Pluralize()`, `UnCapitalize()`)
- **SystemTools.DatabaseToolsShared** — `SqlServerDesignTimeDbContextFactory` base for EF design-time support

## Commands

Build (from repo root):

    dotnet build TravelGuide.slnx

Run (a parameters JSON file is mandatory; it lives outside the repo — see `TravelGuide/Properties/launchSettings.json`):

    dotnet run --project TravelGuide -- --use "D:\1WorkSecurity\TravelGuide\TravelGuide.json"

EF Core migrations (migrations live in `TravelGuideDbMigration`; `TravelGuideFakeHost` is the startup project whose only purpose is design-time `dotnet ef` support):

    dotnet ef migrations add <Name> --project TravelGuideDbMigration --startup-project TravelGuideFakeHost
    dotnet ef database update --startup-project TravelGuideFakeHost

The design-time factory (`TravelGuideFakeHost/TravelGuideDbDesignTimeDbContextFactory.cs`) reads `DataProvider` from FakeHost's `appsettings.json` and the connection string from User Secrets key `ConnectionStringSeed`. `UserSecretsId` is not yet set in `TravelGuideFakeHost.csproj` — run `dotnet user-secrets init` / `dotnet user-secrets set ConnectionStringSeed "..."` there before running migrations against a real database.

There are no test projects.

## Build Strictness

`Directory.Build.props` applies to all projects: `net10.0`, `TreatWarningsAsErrors=true`, `AnalysisMode=All`, SonarAnalyzer.CSharp injected everywhere — any analyzer warning fails the build, so new code must be fully analyzer-clean. `ImplicitUsings` is disabled solution-wide (explicit `using System;` etc. required); a few projects (DbMigration, DbPersistence, FakeHost) re-enable it locally. NuGet versions are centrally managed in `Directory.Packages.props` — `PackageReference` entries in csproj files carry no `Version`.

`.editorconfig` enforces (as errors): file-scoped namespaces, `using` directives outside namespace, braces always, CRLF, 4-space indent. Classes are `sealed` by convention. `//Created by ...Creator` headers mark template-generated files; commented-out blocks (e.g. all of `TravelGuide/TravelGuide.cs`, parts of `MenuData.cs`) are intentional scaffolding examples — leave them unless asked.

## Architecture

Startup flow in `TravelGuide/Program.cs`: parse `--use <parameters.json>` via `ArgumentsParser<TravelGuideParameters>` → build DI container (`TravelGuide/DependencyInjection/TravelGuideServices.cs`) → `CliAppLoop.Run()` renders the main menu built by `TravelGuideMenuBuilder`.

Projects (dependency order):

- **DoTravelGuide** — `TravelGuideParameters`: the persisted app configuration (LogFolder, DatabaseServerConnections, DatabaseParameters, ApiClients, SmartSchemas, plus a `Tasks` dictionary of named `TaskModel`s)
- **TravelGuideDbModels** — EF entity classes (`TestModel` is a sample, deletable)
- **TravelGuideDbPersistence** — `TravelGuideDbContext`. Naming convention in `OnModelCreating`: table = pluralized, uncapitalized class name; columns = uncapitalized property names; unique indexes named `IX_{Tablename}_{columnName}`
- **TravelGuideDbMigration** — migrations assembly (`Migrations/` currently empty)
- **TravelGuideRepoInterfaces / TravelGuideRepositories** — repository layer over the DbContext: `ITravelGuideRepository` (SaveChanges / transaction helpers), `ITravelGuideRepositoryCreatorFactory` (resolves a repository from a fresh DI scope)
- **TravelGuide** — the executable: DI, menu, `TravelGuideTaskRunner`
- **TravelGuideFakeHost** — minimal web host used only as `dotnet ef` startup project
- **TravelGuideDb / LibTravelGuideRepositories** — placeholder projects, no source yet

Database wiring (`TravelGuideServices.AddDatabase`) resolves provider + connection string from the parameters file via `DbConnectionFactory`; only SQL Server is currently wired to `TravelGuideDbContext`.

### Menu system

The main menu is assembled by `CliMenuSetFactory.CreateMenuSet("Main Menu", MenuData.MenuCommandNames, ...)`. To add a main-menu item:

1. Create a class implementing `IMenuCommandFactoryStrategy` that builds a `CliMenuCommand` (example: `TravelGuide/Menu/TravelGuideParametersEdit/TravelGuideParametersEditorListCliMenuCommandFactoryStrategy.cs`).
2. Add its name to `MenuData.MenuCommandNames` (`TravelGuide/Menu/MenuData.cs`).

Strategies are auto-registered in DI by `AddTransientAllStrategies<IMenuCommandFactoryStrategy>` scanning the TravelGuide assembly. Parameters editing UI is `TravelGuideParametersEditor` (field editors per parameter).

### Task pattern (intended crawler workflow)

`TravelGuide/MenuCommands/` contains a complete named-task workflow: `NewTaskCommand` creates a task in the parameters file, `TaskSubMenuCommand` gives each task a submenu (run / rename / delete), and `TaskCommand` runs `TravelGuideTaskRunner` for the selected task (already labeled "Crawler is running..."). These commands are active code but not yet reachable — only the parameters editor is listed in `MenuData.MenuCommandNames`. Wiring tasks into the main menu (via new factory strategies) and implementing `TravelGuideTaskRunner.Run()` are the expected next steps.
